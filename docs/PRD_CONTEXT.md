# 核心项目背景与产品需求文档 (PRD Context)

## 1. 项目元数据
- **项目名称**: Sauna (桑拿)
- **产品定位**: 个人专属的 AI 智囊团协作工作区 (Personal AI Brain Trust Workspace)
- **核心理念**: 通过特定的 Agent 编排与 RAG 技术，复刻各行各业顶尖专家的思维模型。让他们入驻你的专属工作区，陪你一起“蒸桑拿”（沉浸式头脑风暴与战略咨询），为你指引前进的方向。

## 2. 灵感与对标
- **底层技术灵感**: 参考 `nuwa-skill` 等开源架构，采用“思想蒸馏”的概念。利用系统提示词、专业文献、著作等语料库，将大模型微调/约束为特定领域的专家模型（如顶尖科技领袖、资深投资人等）。
- **UI/UX 灵感**: 摒弃传统的“下拉菜单选机器人”列表，采用类似腾讯 Marvis 的多工位可视化的“工位（Workstation）”概念。每个被“蒸馏”出来的专家在屏幕上都有专属卡片，展现“精英团队待命”的视觉冲击力。

## 3. 核心隐喻与术语定义 (术语映射)
*在代码实现和数据库命名时，必须使用括号内的英文技术术语，以符合后端开发规范并确保安全合规；在前端 UI 渲染时，请使用对应的中文文案。*

- **Distill (蒸馏) -> `AgentCreation`**: 创建或配置一个新 AI Agent 的过程。通过注入核心思想库和系统设定，提炼为独立的智囊成员。
- **Workstation (工位) -> `AgentCard`**: 前端 UI 的核心单元。卡片式布局，展示专家头像、姓名、当前思考状态等。
- **Sauna (蒸桑拿) -> `ImmersiveSession`**: 对话互动的过程。象征在无干扰环境中进行激烈的思想碰撞（排汗），提炼真知灼见。

## 4. 核心功能模块与路由规划 (MVP 阶段)

**1. 智囊大厅**
- **前端显示名**: 「智囊大厅」
- **后端模块/路由**: `Lobby` / `/api/v1/lobby`
- **功能**: 展现所有的“工位”。视觉上呈现网格或全景视图，实时显示各位专家处于“空闲”、“沉思”或“对话中”的状态。

**2. 蒸馏车间**
- **前端显示名**: 「蒸馏车间」
- **后端模块/路由**: `Studio` / `/api/v1/studio`
- **功能**: 用于新建和配置专家的页面。需要配置项：角色名称、系统设定（System Prompt）、知识库挂载（上传 PDF 自传/文献至 RAG 系统）、语气设定。

**3. VIP 桑拿房**
- **前端显示名**: 「VIP 桑拿房」
- **后端模块/路由**: `FocusRoom` / `/api/v1/focus-room`
- **功能**: 与某位指定专家的深度 1v1 对话界面。极简、专注，体现“高端咨询”氛围。

**4. 董事会桑拿 (进阶功能)**
- **前端显示名**: 「董事会桑拿」
- **后端模块/路由**: `BoardMeeting` / `/api/v1/board-meeting`
- **功能**: 抛出一个问题，勾选多位专家，让他们基于各自的认知框架为你提供多维度的分析或互相辩论。

## 5. 技术栈与架构约束 (Technical Stack & Architecture)
你现在是 Sauna 项目的首席架构师兼全栈工程师。在编写任何代码之前，必须严格遵守以下技术规范：

- **Frontend**: 
  - 框架：Next.js 16 (App Router) + React 19。
  - 样式：TailwindCSS。
  - 状态：Zustand。
  - 通信：使用 SSE (Server-Sent Events) 处理大模型的流式对话输出。
  - **重要原则：所有的组件命名使用英文，但在渲染用户看到的页面标题和按钮时，必须严格使用上述第 4 节规定的中文“前端显示名”。**
- **Backend**: 
  - 语言与框架：Golang (建议 Gin 或 Hertz)。
  - 架构规范：必须严格遵循 Clean Architecture (Domain -> Repository -> Usecase -> Handler)。
  - 关注点分离：AI 模型调度逻辑、提示词组装、以及大模型 API 调用逻辑必须与 HTTP 层完全解耦，以支持灵活切换不同的模型 Base URL。
- **Database**: 
  - 关系型存储：PostgreSQL (配合 GORM 或 sqlc) 存储用户、会话、Agent 配置等。
  - 向量知识库：必须启用 PostgreSQL 的 `pgvector` 扩展，用于 Agent 蒸馏语料的 Embedding 存储和相似度检索。禁止引入外部的第三方向量数据库。
